# ラップカメラ

陸上の練習・大会で、動画を撮りながら通過タイム（合計・区間）を画面に表示・記録するためのWebアプリ。

- 録画ボタン … 撮影の開始/停止
- ラップボタン … 400m通過などで押すと、その時点の合計タイム・区間タイムを記録
- 通過タイム一覧 … 画面左に常時表示（最新が大きく黄色で表示）
- 動画には TOTAL と現在ラップが焼き込まれる（タイム一覧は付き添い用で動画には入らない）

## GitHub Pages での公開手順

スマホのカメラを使うには **HTTPS** で開く必要があります。GitHub Pages なら無料でHTTPS公開でき、編集するたびに自動更新されます。

### 1. リポジトリを作る
1. https://github.com にログイン（アカウントがなければ作成）
2. 右上の「+」→「New repository」
3. Repository name に好きな名前（例: `lap-camera`）
4. **Public** を選ぶ（無料でPagesを使うため）
5. 「Create repository」

### 2. ファイルを置く
1. 作ったリポジトリのページで「Add file」→「Upload files」
2. この `index.html` をドラッグ＆ドロップ（`README.md` も一緒に上げてOK）
3. 下の「Commit changes」を押す

> ※ファイル名は必ず `index.html`。Pages はこれを自動でトップページとして表示します。

### 3. Pages を有効化
1. リポジトリの「Settings」→ 左メニュー「Pages」
2. Source を「Deploy from a branch」
3. Branch を `main`、フォルダを `/ (root)` にして「Save」
4. 1〜2分待つと、上部に公開URLが出ます
   → `https://ユーザー名.github.io/lap-camera/`

### 4. スマホで開いてアプリ化
1. iPhone の **Safari** でそのURLを開く（Chromeより確実）
2. カメラの使用を「許可」
3. 共有ボタン → 「ホーム画面に追加」
   → アプリのアイコンから全画面で起動できます

---

## あとから機能を追加・修正するには
1. GitHub のリポジトリで `index.html` を開き、鉛筆アイコン（Edit）をタップ
2. 直して「Commit changes」
3. 1分ほどで公開URLに自動反映（反映されないときはページを再読み込み）

慣れてきたら、PCに `git clone` してローカルで編集→`git push` する方法も使えます。

## メモ
- 録画中は画面が暗くならないようにしています（Wake Lock）。
- 動画の保存は終了画面の「動画を保存」→ 共有から「ビデオを保存」で写真アプリへ。
- 長時間録画はスマホのメモリを使うため、1本あたり数分を目安に。

## 将来 App Store に出す場合
このWeb版でフォーム/機能を固めてから、ネイティブ化（Swift で作り直す、または Capacitor 等でラップする）を検討するのがおすすめ。仕様が固まっているほど移行が楽になります。
